**Honest local-LLM verdict:** chat, tool-calling, guardrails, sandbox, and the connection self-test all work end-to-end on a local model with zero cloud calls and `cost=0`. **Structured/typed output does *not* reliably work on tiny local models (0.5b/1b)** тАФ the model emits the JSON *schema* (or malformed JSON) instead of an instance, and the prompt-fallback parser can't recover it. This is a model-capability issue (capable models тАФ DeepSeek, Gemini тАФ pass structured output), but it means the "typed agents" wedge should be documented as **requiring a capable model**; do not imply 1b-class local models give reliable typed output.

> Config note discovered live: `LARGESTACK_OLLAMA_OPENAI_COMPAT=1` (and all `LARGESTACK_*` env) is read by a **cached** pydantic-settings singleton (`_core/config.py:99-102`) тАФ it must be set **before** the first `get_config()` (i.e. before the first `Agent`). Worth a docs line.
